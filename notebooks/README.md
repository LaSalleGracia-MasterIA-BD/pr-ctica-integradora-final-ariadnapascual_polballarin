# Notebooks del proyecto laSalle Health Center

Estos notebooks están pensados para ejecutarse desde la raíz del repositorio o desde la propia carpeta `notebooks/`.

Orden recomendado:

1. `00_overview_sistema.ipynb` — arquitectura global.
2. `01_pipeline_bigdata.ipynb` — pipeline, Dask y calidad.
3. `02_ml_triage_tabular.ipynb` — modelos tabulares.
4. `03_ml_inference_radiografias.ipynb` — radiografías, comparación de modelos y criterio clínico.
5. `04_monitoring_automation.ipynb` — Loki, Promtail, eventos y alertas.
6. `05_dashboard_api_demo.ipynb` — API, dashboard y demo.

Los notebooks no sustituyen al dashboard. Sirven para explicar gráficamente el proyecto, enseñar métricas, matrices de confusión, decisiones técnicas y limitaciones.
